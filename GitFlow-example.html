GitFlow

  • Создать репозиторий на GitHub и клонировать его на компьютер.
  • Создаем ветку разработки - git branch develop
  • Переходим на ветку разработки - git checkout develop
  • Отправляем ветку на GitHub - git push origin develop
  • Создать от ветки develop feature/-ветки и мерджить feature/-ветки в develop (release/0.1.0), когда фичи будут выполнены
  • Создание ветки release/0.1.0 от develop - git branch develop release/0.1.0 и Переходим на ветку разработки - git checkout release/0.1.0 (git branch - список веток и на какой находимся)
  • Все feature/-ветки по окончанию работы мерджатся в ветку release/0.1.0
  • Когда ветка release/0.1.0 закончена, то она мерджится в develop и main и затем удаляется
  • Если в ветке main обнаруживается ошибка, то создается hotfix-ветка - по окончании работы hotfix-ветка мерджится в ветку develop и ветку main
  • git checkout develop - git merge hotfix - git push origin develop
  • git checkout main - git merge hotfix - git push origin main
  • После этого hotfix-ветка удаляется (локально) - git branch -d hotfix
  • После этого hotfix-ветка удаляется (на GitHub) - находясь в ветке develop нажимаем кнопка branches и удаляем ветку hotfix
